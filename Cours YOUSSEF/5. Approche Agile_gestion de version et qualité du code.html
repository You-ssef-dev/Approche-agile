🔀 Git & GitLab

Gestion de versions distribuée et collaboration

📌 Section 1

📂 Gestion de Versions (VCS)

Stocker, suivre et restaurer les différentes versions d'un projet.

💻 Local

Copies locales des fichiers. Simple mais risqué (pas de partage).

🖥️ Centralisé (SVN)

Un serveur central stocke le dépôt. Nécessite une connexion réseau.

🌐 Distribué (Git)

Chaque développeur possède une copie complète du projet. Travail hors ligne possible.

💡 Autres outils : CVS, SVN, Mercurial, TFS (Microsoft). Git est devenu le quasi-standard.

📌 Section 3

⌨️ Commandes Git Principales

Les commandes essentielles pour travailler avec Git.

Commande Description
git init Créer un nouveau dépôt vide
git status Afficher les différences entre les zones
git add Ajouter des fichiers à l'index (staging)
git commit -m "msg" Sauvegarder une version dans HEAD
git log Afficher l'historique des commits
git clone Cloner un dépôt existant (local ou distant)
git pull Récupérer les modifications depuis un dépôt distant
git push Publier les modifications vers un dépôt distant

📝 Exemple pratique

# 1. Initialiser un projet mkdir mon_projet && cd mon_projet git init # 2. Créer et ajouter un fichier echo "Bonjour" > file.txt git add file.txt # 3. Valider le changement git commit -m "Premier commit" # 4. Voir l'historique git log --oneline
📌 Section 4

🌿 Gestion des Branches

Développer des fonctionnalités isolées et les fusionner ensuite.

🌿 Créer & Basculer

# Créer une branche git branch testing # Basculer vers la branche git checkout testing git switch testing # Créer + basculer en une commande git checkout -b new-branch git switch -c new-branch

🔀 Fusionner (Merge)

# Revenir sur master git checkout master # Fusionner la branche git merge testing # Supprimer la branche git branch -d testing

📌 Section 11

🔗 Intégration ALM

SonarQube dans le cycle de vie du développement logiciel.

🔄 Workflow ALM complet

1Développeurs codent dans IDE + SonarLint (analyse locale temps réel)
2Push vers SCM (GitHub, GitLab…)
3CI/CD déclenche automatiquement le SonarScanner
4Rapport envoyé au serveur SonarQube
5Résultats stockés en base de données et affichés sur le dashboard
6Développeurs review et corrigent la dette technique
7Managers reçoivent les rapports d'analyse
💡 SonarLint est une extension IDE gratuite qui identifie les problèmes en temps réel. Compatible VSCode, Eclipse, JetBrains…
