📌 Section 5

📦 Exemple : La Classe Carre

Un exemple concret pour comprendre les tests unitaires.

📄 Carre.php (src/)

// Classe à tester class Carre { private $cote; public function setCote($cote) { if ($cote < 0) throw new Exception("Valeur invalide!"); $this->cote = $cote; } public function surface() { return $this->cote * $this->cote; } }

🧪 CarreTest.php (tests/)

// Classe de test class CarreTest extends TestCase { public function testSurface() { $objet = new Carre(10); $this->assertEquals( 100, $objet->surface() ); } }
✅ assertEquals compare le résultat attendu (100) au résultat obtenu. Si égal → test réussit. Si inégal → test échoue.
📌 Section 6

▶️ Lancement des Tests

Exécuter et interpréter les résultats de PHPUnit.

▶️ Commande d'exécution

# Lancer tous les tests vendor/bin/phpunit tests # Avec couleurs vendor/bin/phpunit tests --colors # Lancer un seul test vendor/bin/phpunit --filter testSurface

📋 Résultat

PHPUnit 9.5.26 by Sebastian Bergmann and contributors. . 1 / 1 (100%) Time: 00:00.022, Memory: 4.00 MB OK (1 test, 1 assertion)

Le résultat indique : nombre de tests, temps d'exécution, et status (OK ou Failure).

📌 Section 7

⚠️ Tester les Exceptions

Utiliser expectException pour vérifier qu'une exception est bien levée.

⚠️ expectException

public function testSetCote() { $this->expectException(Exception::class); $objet = new Carre(-23); }

Le test réussit si l'exception est levée. Il échoue si aucune exception n'est lancée.

🔴 Ici, le test réussit car setCote(-23) lance bien une exception pour une valeur négative.
📌 Section 8

📊 Data Providers

Tester plusieurs valeurs avec une seule méthode grâce aux fournisseurs de données.

📊 @dataProvider

/** * @dataProvider dataForTestSurface */ public function testSurface($cote, $resultatAttendu) { $object = new Carre(); $object->setCote($cote); $this->assertEquals($resultatAttendu, $object->surface()); } public function dataForTestSurface() { return [ [0, 0], [10, 100], [5, 25] ]; }
1

$cote = 0
Attendu: 0

2

$cote = 10
Attendu: 100

3

$cote = 5
Attendu: 25

✅ PHPUnit exécute le test 3 fois automatiquement. Résultat : OK (3 tests, 3 assertions)
📌 Section 9

📊 Couverture de Code

Mesure du pourcentage de code source exécuté lors des tests.

📊 Code Coverage

Pourcentage de code exécuté pendant les tests. Plus la couverture est haute, moins de bugs non détectés.

⚠️ Important

SonarQube ne génère PAS le rapport de couverture. PHPUnit génère le rapport, SonarScanner l'envoie à SonarQube.

📌 Section 10

⚙️ Configuration Xdebug

Installer et configurer Xdebug pour générer les rapports de couverture.

1

2

3

Ajouter les lignes Xdebug

📝 Lignes à ajouter dans php.ini

# Sous la section [XDebug] zend_extension=xdebug xdebug.mode=coverage xdebug.start_with_request=trigger # Désactiver output buffering output_buffering = Off

Puis redémarrer Apache.

📌 Section 11

📄 Générer les Rapports

Produire les fichiers XML de couverture et d'exécution des tests.

▶️ Commande de génération

XDEBUG_MODE=coverage vendor/bin/phpunit \ --coverage-clover=phpunit-coverage-result.xml \ --log-junit=phpunit-execution-result.xml

📝 sonar-project.properties

# Clé du projet sonar.projectKey=monProjet sonar.projectBaseDir=. # Exclusions sonar.exclusion=vendor/** sonar.coverage.exclusions=tests/* # Chemins des rapports sonar.php.coverage.reportPaths=phpunit-coverage-result.xml sonar.php.tests.reportPath=phpunit-execution-result.xml
📌 Section 12

🔎 Lancer SonarScanner

Envoyer les rapports au serveur SonarQube.

🚀 Commande SonarScanner

sonar-scanner.bat \ -D"sonar.projectKey=monProjet" \ -D"sonar.sources=." \ -D"sonar.host.url=http://localhost:9000" \ -D"sonar.login=VOTRE_TOKEN"
