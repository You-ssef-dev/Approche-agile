## 📖 Description

This project presents the **Agile Approach** including:

- ✅ The Agile Manifesto (4 values & 12 principles)
- ✅ Agile methodologies (Scrum, XP...)
- ✅ Product Backlog organization
- ✅ User Stories & Epics
- ✅ Scrum roles
- ✅ Scrum events
- ✅ Software Increment & Sprint delivery

---

## 🚀 1️⃣ Manifeste Agile

### 🔹 The 4 Agile Values

1. Individuals and interactions over processes and tools  
2. Working software over comprehensive documentation  
3. Customer collaboration over contract negotiation  
4. Responding to change over following a plan  

---

### 🔹 The 12 Agile Principles

✔ Customer satisfaction  
✔ Welcome changing requirements  
✔ Deliver working software frequently  
✔ Business & developers collaboration  
✔ Build projects around motivated individuals  
✔ Face-to-face communication  
✔ Working software as progress measure  
✔ Sustainable development  
✔ Technical excellence  
✔ Simplicity  
✔ Self-organizing teams  
✔ Regular reflection & improvement  

---

## 🛠️ 2️⃣ Agile Methodologies

### 🔸 Scrum
- Framework based on sprints
- Iterative & incremental delivery

### 🔸 XP (Extreme Programming)
- Focus on code quality
- Continuous testing
- Pair programming

---

## 📋 3️⃣ Product Backlog

A prioritized list of:

- User Stories
- Features
- Improvements
- Bug fixes

Organized and managed by the **Product Owner**

---

## 👥 4️⃣ Scrum Roles

| Role | Description |
|------|-------------|
| 👤 Product Owner | Defines product vision & backlog |
| 🧑‍💼 Scrum Master | Facilitates Scrum process |
| 👨‍💻 Development Team | Delivers the increment |

---

## 📅 5️⃣ Scrum Events

- 📝 Sprint Planning  
- 🌅 Daily Stand-up (15 min)  
- 🔎 Sprint Review  
- 🔄 Sprint Retrospective  

---

## 📦 6️⃣ Software Increment

At the end of each sprint:

- ✔ A working product increment
- ✔ Potentially shippable
- ✔ Tested and validated

Agile methodology improves:

- ⚡ Productivity  
- 🔄 Flexibility  
- 🤝 Collaboration
